Tencent Cloud Cloud Virtual Machine(CVM)을 추가할 때 사용자 지정 인스턴스 유형이 인스턴스의 CVM 하드웨어 사양을 결정합니다. 인스턴스 유형마다 서로 다른 컴퓨팅, 메모리 및 스토리지 기능을 제공합니다. 사용자는 배포 실행할 애플리케이션 규모를 기반으로 적당한 인스턴스 유형을 선택할 수 있습니다. 이런 인스턴스 패밀리는 CPU, 메모리, 스토리지, 이종 하드웨어와 네트워크 대역폭의 구성으로 서로 다르게 조합되어 있어, 사용자의 애플리케이션에 적합한 리소스를 자유롭게 선택할 수 있습니다.

갑작스러운 서비스 피크에 대비해, 사용자는 종량제 과금 방식을 선택하여 컴퓨팅 인스턴스를 수시로 활성화/폐기하고 인스턴스의 실제 사용량에 따라 결제할 수 있습니다. 과금은 초 단위까지 정확하게 카운팅 되어 원가를 최대한 낮출 수 있습니다.

용어 설명

  • 내부 네트워크 대역폭 기능: 인스턴스 내부 네트워크의 초당 전송 가능한 최대 데이터양(bit)

  • 멀티 큐 기술: 버츄얼 ENI는 여러 개의 송수신 패킷 큐를 지원하며 각 큐에 있는 데이터 패킷을 통해 서로 다른 CPU가 동시에 송수신 프로세싱을 진행할 수 있습니다.

  • 큐 개수: 각 버츄얼 ENI가 지원하는 송수신 패킷 큐의 수량(N 큐는 N개 수신 큐와 N개 송신 큐의 동시 지원을 표시함)

인스턴스 제한

  • 한 구역에서 시작할 수 있는 총 인스턴스 수는 제한되어 있습니다. 제한 관련하여 보다 많은 정보는 CVM 인스턴스 구매 제한을 참고 바랍니다.

  • 인스턴스의 시스템 디스크와 데이터 디스크의 마운트 제한: 디스크의 IO 성능 체험을 보장하기 위해 Tencent Cloud는 인스턴스와 함께 구매한 데이터 디스크의 일부 사이즈와 유형에 제한을 두고 있으니, 인스턴스 패밀리에 대응하는 디스크 매칭 모듈을 참고하시길 바랍니다. 필요 시 독립된 Cloud Block Storage(CBS)를 구매하여 디스크 제한을 피할 수 있습니다.

  • 인스턴스 사양에 대응하는 내부 네트워크 대역폭 기능에 주목하길 바랍니다. 해당 지표는 대응하는 사양 인스턴스의 최대 내부 네트워크 대역폭 최댓값입니다. CVM 내부 네트워크 트래픽이 최댓값을 초과하면 인스턴스 내부 네트워크에 랜덤 패킷 손실이 발생할 수 있습니다.

  • 지역에 따라 판매 가능한 인스턴스 사양에 차이가 있고 일부 사양은 품절되었을 수 있으며, 실제 구매 페이지에 있는 정보를 기준으로 합니다.

  • 풀 텍스트에서의 네트워크 송수신 패킷은 순수 네트워크 포워딩 테스트 시나리오에서의 데이터로, 테스트 방법은 네트워크 성능 테스트를 참고할 수 있습니다. 서비스 시나리오에서의 성능 데이터는 반드시 실제로 테스트한 후 용량을 다시 예상해야 합니다.

표준형 인스턴스 패밀리

표준형 인스턴스는 컴퓨팅, 메모리 및 네트워크 자원이 균형을 이루어 다수 시나리오에서의 애플리케이션 자원의 수요를 만족시킬 수 있습니다.

표준형 S5

표준형 S5 인스턴스는 시리즈3의 표준형 인스턴스로, 신규 최적화된 버츄얼화 플랫폼에 기반하여 균형 잡힌 안정적인 컴퓨팅, 메모리 및 네트워크 자원을 제공하며 수많은 애플리케이션의 최상의 선택입니다.

표준형 S5 인스턴스는 Xeon® 프로세서 Cascade Lake 최신 프로세서, 최신 DDR4 메모리를 탑재하고 있고 기본으로 네트워크 최적화가 되어 있으며, 내부 네트워크 송수신 능력은 최고600만pps에 달하고 최고 내부 네트워크 대역폭은 28Gbps를 지원 가능합니다.

인스턴스 특징

  • 2.5GHz Intel® Xeon® Cascade Lake 프로세서, turbo 3.1GHz, 시리즈 3 6채널 DDR4, 메모리 안정적인 컴퓨팅 성능

  • 새로운 Intel Advanced Vector Extension (AVX-512) 명령어 집합을 갖추고 있습니다.

  • 프로세서와 메모리의 배분 비율은 1:2, 1:4입니다.

  • 최대 28Gbps의 내부 네트워크 대역폭을 지원하며 초고속 네트워크 송수신 패킷 능력으로 아주 높은 내부 네트워크 전송 요구 사항을 충족시킬 수 있습니다.

  • 인스턴스의 네트워크 성능은 사양에 비례하며 사양이 높을수록 네트워크 포워딩 성능이 더 뛰어나며 내부 네트워크 대역폭의 최댓값이 더 큽니다.

사용 시나리오

표준형 S5 인스턴스는 다음의 시나리오에 응용할 수 있습니다.

  • 다양한 유형과 규모의 기업급 애플리케이션

  • 중소형 데이터베이스 시스템, 캐시, 검색 클러스터

  • 컴퓨팅 클러스터, 메모리에 의존하는 데이터 처리

  • 비디오 댓글 자막, 라이브 방송, 게임 등과 같은 하이 네트워크 패킷 송수신 시나리오

인스턴스 요구

  • S5 인스턴스는 정액 과금제 인스턴스와 종량제 인스턴스로 사용할 수 있습니다.

  • 사설 네트워크에서 시작하는 S5 인스턴스만 지원합니다.

  • S5 인스턴스가 지원하는 구매 사양은 다음의 인스턴스 사양을 참고 바랍니다. 선택한 S5 인스턴스의 사이즈가 사용자의 운영 체제와 애플리케이션의 최저 CPU 메모리 요구를 만족시킬 수 있어야 합니다. 많은 사용 사례 중, 대량의 메모리와 CPU 자원을 소모하는 그래픽 사용자 인터페이스를 가진 운영 체제(예: Windows)는 더 큰 인스턴스 사이즈를 필요로 할 수 있습니다. 사용자의 워크로드가 메모리와 CPU에 요구하는 수치가 시간이 지날수록 높아지므로, 사용자는 더 높은 사양으로 변경하거나 다른 유형의 인스턴스를 선택하여 사용할 수 있습니다.

표준형 SA2

표준형 SA2 인스턴스는 AMD EPYC™ 프로세서를 적용하여 시리즈 3 인스턴스로 여러 가지 사양을 제공하고 좋은 가성비를 자랑하므로 사용자의 워크로드가 업계에서 선두적인 가성비를 얻을 수 있도록 보장합니다. 해당 인스턴스는 균형 잡힌 컴퓨팅, 메모리 및 네트워크 자원을 제공함으로써 수많은 애플리케이션의 최상의 선택입니다.

표준형 SA2 인스턴스는 AMD EPYC™ ROME 최신 프로세서, 최신 DDR4 메모리를 탑재하고 있고 기본으로 네트워크 최적화가 되어 있으며, 내부 네트워크 송수신 능력은 최고 750만pps에 달하고 내부 네트워크 대역폭은 최고 25Gbps까지 지원 가능합니다.

설명:

표준형 SA2 인스턴스는 베타 테스트 단계에 있습니다. 이곳을 클릭하여 SA2 인스턴스 베타 테스트 자격 신청을 제출하여 주십시오.

인스턴스 특징

  • 2.6GHz AMD EPYC™ ROME 프로세서, turbo 3.3GHz, 시리즈 3 8채널 DDR4 탑재, 메모리 컴퓨팅 성능이 안정적입니다.

  • 보다 큰 인스턴스 사양, SA2.45XLARGE464, 180 vCPU와 464GB 메모리를 제공합니다.

메모리형 M5

메모리형 M5 인스턴스는 시리즈 3 메모리형 인스턴스로, 신규 최적화된 버츄얼화 플랫폼에 기반하여 메모리 안의 대형 데이터 세트를 처리하기 위한 워크로드에 빠르고 안정적인 성능을 제공하는 데 목적을 둠으로써, 하이 메모리 컴퓨팅 애플리케이션의 최상의 선택입니다.

메모리형 M5 인스턴스는 Xeon® 프로세서 Cascade Lake 최신 프로세서, 최신 DDR4 메모리를 탑재하고 있고 기본으로 네트워크 최적화가 되어 있으며, 내부 네트워크 송수신 능력은 최고 600만pps에 달하고 내부 네트워크 대역폭은 최고 25Gbps까지 지원 가능합니다.

사용 시나리오

  • DNA 컴퓨팅 등 Hadoop 클러스터 또는 Redis를 자체로 구축하는 사용자

  • 비디오 댓글 자막, 라이브 방송, 게임 등과 같은 하이 네트워크 패킷 송수신 시나리오

메모리형 M1

메모리형 M1 은 CPU와 메모리의 비율이 약 1:8 인 모델이며 고성능 데이터베이스, 분산식 메모리 캐시 등 대량의 메모리 작업, 검색 및 컴퓨팅이 필요한 애플리케이션의 수요를 만족시킬 수 있습니다.

인스턴스 특징

  • 2.3GHz Intel Xeon® E5-2670 v3 프로세서, DDR3 메모리, 사용자에게 보다 큰 인스턴스 사양을 제공하며 전반적인 컴퓨팅 능력이 더 강합니다.

  • 메모리 집약형 애플리케이션의 황금비율, 빅 서비스 배포 수요를 만족시킬 수 있습니다.

사용 시나리오

다음과 같은 상황에 아주 적합합니다.

  • 대량의 메모리 작업, 검색 및 컴퓨팅이 필요한 고성능 데이터베이스, 분산식 메모리 캐시 등의 애플리케이션

  • DNA 컴퓨팅 등 Hadoop 클러스터 또는 Redis를 자체로 구축하는 사용자

인스턴스 요구

  • M1 인스턴스는 정액 과금제 인스턴스와 종량제 인스턴스로 사용할 수 있습니다.

  • 기본 네트워크와 사설 네트워크에서 시작하는 M1 인스턴스를 지원합니다.

  • M1 인스턴스가 지원하는 구매 사양은 다음의 인스턴스 사양을 참고 바랍니다.

고 IO 형 인스턴스 패밀리

고 IO 형 인스턴스는 높은 랜덤 IOPS, 높은 처리량, 액세스 짧은 딜레이 등의 특징을 갖추고 있어 디스크의 읽기/쓰기와 딜레이 시간에 대한 요구가 높은 고성능 데이터베이스 등 I/O 집약형 애플리케이션에 적합합니다.

고 IO 형 IT3

고 IO 형 IT3 인스턴스는 I/O 집약형 워크로드를 위해 전문 설계한 것입니다. NVMe SSD 인스턴스 스토리지에 기반하여 짧은 딜레이, 극히 높은 IOPS, 높은 처리량을 갖춘 스토리지 자원을 제공하고 180만 IOPS 및 11GB/s의 높은 처리 능력을 제공하며 낮은 비용으로 아주 높은 IOPS를 제공하고 있습니다. 고성능 관계형 데이터베이스, Elasticsearch 등 IO 집약형 서비스에 적합합니다.

IT3 인스턴스는 저렴하고 집약된 스토리지를 제공하며 각 CPU 핵심에 높은 IOPS 와 더 많은 내부 네트워크 대역폭을 제공할 수 있습니다.

설명:

고 IO 형 IT3 인스턴스는 일시적으로 화이트리스트 오픈 상태에 처해 있어, 사용자의 프리세일즈 매니저에게 연락하여 해당 인스턴스를 구매하는 권한을 활성화해 주시기 바랍니다.

사용 시나리오

  • 고성능 데이터베이스, NoSQL 데이터베이스(예: MongoDB), 클러스터화 데이터베이스

  • 온라인 트랜잭션 처리(OLTP) 시스템, Elastic Search 검색 등 짧은 딜레이 시간이 필요한 I/O 집약형 애플리케이션

인스턴스 특징

  • 2.5 GHz Intel® Xeon® Skylake 6133 프로세서, 컴퓨팅 성능이 안정적입니다.

  • 시리즈 3 6채널 DDR4 메모리

  • 최대 23Gbps의 내부 네트워크 대역폭을 지원할 수 있으며 아주 높은 내부 네트워크 전송 요구 사항을 충족시킬 수 있습니다.

  • NVMe SSD의 인스턴스 스토리지를 적용하고 짧은 딜레이, 아주 높은 IOPS를 제공합니다.

    • 싱글 디스크의 랜덤 읽기 성능은 62만 IOPS(4KB Block Size)에 달하고 순서 읽기 처리 능력은 2.9GB/s(128KB Block Size)에 달합니다.

    • 기기 전체의 랜덤 읽기 성능은 180만 IOPS(4KB Block Size)에 달하고 순서 읽기 처리 능력은 11GB/s(128KB Block Size)에 달합니다.

인스턴스 요구

  • IT3 인스턴스의 데이터 디스크는 로컬 스토리지로 데이터 분실의 리스크(예를 들어 호스트다운 시)가 있습니다. 만일 사용자의 애플리케이션이 데이터 신뢰성을 보장하는 아키텍처를 구축할 수 없다면 IT3 인스턴스의 사용은 비추천드리며, 데이터 디스크로 CBS를 선택할 수 있는 인스턴스를 사용할 것을 강력히 권장합니다.

    • 로컬 디스크가 훼손되면 사용자가 CVM 비활성화 작업을 진행한 후에야 저희가 로컬 디스크의 교체 작업을 할 수 있습니다.

    • 만일 CVM 인스턴스가 이미 다운되었다면 저희는 이를 사용자에게 알리고 보수 작업을 진행합니다.

  • IT3 인스턴스는 정액 과금제 인스턴스와 종량제 인스턴스로 사용할 수 있습니다.

  • 사설 네트워크에서 시작하는 IT3 인스턴스만 지원합니다.

  • IT3 인스턴스는 사양의 조절을 지원하지 않습니다.

  • IT3 인스턴스가 지원하는 구매 사양은 다음의 인스턴스 사양을 참고 바랍니다.

빅 데이터형 인스턴스 패밀리

빅 데이터형 인스턴스는 대량 스토리지 자원을 탑재하고 높은 처리 능력을 갖추고 있어 Hadoop 분산식 컴퓨팅, 대량의 로그 처리, 분산식 파일 시스템 및 대형 데이터 웨어하우스 등 처리 집약형 애플리케이션에 적합합니다.

빅 데이터형 D2

빅 데이터형 D2 인스턴스는 시리즈 3의 빅 데이터형 인스턴스로 고처리, 대량의 스토리지 자원을 갖추었고 최고로 144T SATA HDD 로컬 스토리지를 탑재할 수 있으며 Hadoop 분산식 컴퓨팅, 병렬 데이터 프로세싱 등 처리 집약형 서비스 사용에 적합합니다.

사용 시나리오

  • Hadoop MapReduce/HDFS/Hive/HBase 등 분산식 컴퓨팅

  • Elasticsearch, 로그 처리 및 대형 데이터 웨어하우스 등 서비스 시나리오 설계

  • 인터넷 업종, 금융 업종 등 빅 데이터 컴퓨팅과 스토리지 분석 수요가 있는 업종의 고객, 대량의 데이터 저장과 컴퓨팅을 진행하는 서비스 시나리오

인스턴스 특징

  • 2.4GHz Intel® Xeon® Skylake 6148 프로세서, DDR4 메모리

  • 인스턴스는 최대 12개의 12TB 로컬 디스크를 탑재하고 최고 144TB의 HDD에 기반한 로컬 스토리지를 갖춥니다.

    • 싱글 디스크의 순서 읽기 처리 능력은 220+MB/s이고 순서 쓰기 처리 능력은 220+MB/s(128KB Block Size, 32 Depth)입니다.

    • 기기 전체의 처리 능력은 최고 2.8GB/s(128KB Block Size, 32 Depth)에 도달할 수 있습니다.

    • 2ms - 5ms로 낮은 읽기/쓰기 딜레이 시간

  • 보다 큰 인스턴스 사양, D2.19XLARGE320, 최고 76vCPU와 320GB 메모리를 제공합니다.

  • 로컬 스토리지의 단가는 S2 의 1/10 정도로 저렴하며, IDC에서 Hadoop 클러스터를 자체 구축하는 것과 비용이 비슷합니다.

  • 프로세서와 메모리의 배분 비율은 1:4이며 빅 데이터 시나리오 설계입니다.

인스턴스 요구

  • 빅 데이터형 D2 인스턴스의 데이터 디스크는 로컬 디스크로 데이터 분실의 리스크(예를 들어 호스트다운 시)가 있습니다. 만일 사용자의 애플리케이션이 데이터 신뢰성을 보장하는 아키텍처를 완성할 수 없다면 사용자가 데이터 디스크로 CBS를 선택할 수 있는 인스턴스의 사용하기를 강력히 권장합니다.

  • 빅 데이터형 D2 인스턴스는 정액 과금제 인스턴스와 종량제 인스턴스로 사용할 수 있습니다.

컴퓨팅형 인스턴스 패밀리

컴퓨팅형 인스턴스 패밀리는 3.2GHz 기준 클록 속도를 탑재하고 최고 싱글 코어 컴퓨팅 성능을 갖고 있습니다. 배치 프로세싱, 고성능 컴퓨팅 및 대형 게임 서버 등 컴퓨팅 집약형 애플리케이션에 적합합니다.

컴퓨팅형 CN3

컴퓨팅형 CN3 인스턴스는 시리즈 3 컴퓨팅형 인스턴스로 내부 네트워크 대역폭은 최고 25Gbps까지에 도달 가능하고 더 큰 대역폭과 더 낮은 딜레이 시간을 보유하고 있습니다. CVM 중 최고 기준 클록 속도의 프로세서와 최고의 가성비를 제공하며 하이 컴퓨팅 성능과 하이 동시 발생 읽기/쓰기 등 컴퓨팅 제한을 받는 애플리케이션의 이상적인 선택입니다.

컴퓨팅형 CN3 인스턴스는 Xeon® 프로세서 Skylake 최신 프로세서를 탑재하고 있으며 내부 네트워크 대역폭은 최고 25Gbps까지 지원 가능하여 컴퓨팅형 C3 보다 2.5배 향상되었습니다.

사용 시나리오

다음과 같은 상황에 아주 적합합니다.

  • 배치 프로세싱 워크로드, 고성능 컴퓨팅(HPC)

  • 하이 트래픽 Web 프런트 엔드 서버

  • 대규모 다중사용자 온라인(MMO) 게임 서버 등 기타 컴퓨팅 집약형 서비스

인스턴스 특징

  • 3.2GHz Intel® Xeon® Skylake 6146 프로세서

  • 새로운 Intel Advanced Vector Extension (AVX-512) 명령어 집합을 갖추고 있습니다.

  • 최고 25Gbps의 내부 네트워크 대역폭을 지원할 수 있으며 아주 높은 내부 네트워크 전송 요구 사항을 충족시킬 수 있습니다.

  • 시리즈 3 6채널 DDR4 메모리, 메모리 대역폭이 2666MT/s에 달합니다.

  • 프로세서와 메모리의 배분 비율은 1:2, 1:4입니다.

  • 인스턴스의 네트워크 성능은 사양에 비례하며 사양이 높을수록 네트워크 포워딩 성능이 더 뛰어나며 내부 네트워크 대역폭의 최댓값이 더 큽니다.

  • 모든 종류의 CBS를 지원합니다.

인스턴스 요구

Tencent Cloud의 최신 버츄얼화를 적용하면 컴퓨팅 성능의 소모가 없고 중첩 버츄얼화를 지원합니다. Hadoop 분산식 컴퓨팅, 병렬 데이터 프로세싱 등 처리 집약형 서비스 사용에 적합합니다.

사용 시나리오

  • Hadoop MapReduce/HDFS/Hive/HBase 등 분산식 컴퓨팅

  • Elasticsearch, 로그 처리 및 대형 데이터 웨어하우스 등 서비스 시나리오 설계

  • 인터넷 업종, 금융 업종 등 빅 데이터 컴퓨팅과 스토리지 분석 수요가 있는 업종의 고객, 대량의 데이터 저장과 컴퓨팅을 진행하는 서비스 시나리오

인스턴스 특징

  • 2.4 GHz Intel® Xeon® Skylake 6148 프로세서, DDR4 메모리

  • 인스턴스는 최대 12개의 12TB 로컬 디스크를 탑재하고 최고 144TB의 HDD에 기반한 로컬 스토리지를 갖춥니다.

  • 싱글 디스크의 순서 읽기 처리 능력은 220+MB/s이고 순서 쓰기 처리 능력은 220+MB/s(128KB Block Size, 32 Depth)입니다.

  • 기기 전체의 처리 능력은 최고 2.8GB/s(128KB Block Size, 32 Depth)에 도달할 수 있습니다.

  • 2ms - 5ms로 낮은 읽기/쓰기 딜레이 시간
