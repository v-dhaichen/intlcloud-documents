### 도메인 이름을 어떻게 액세스합니까?
CDN 콘솔에서 도메인 이름을 액세스 할 수 있습니다. 자세한 정보는 [도메인 이름 액세스](https://intl.cloud.tencent.com/document/product/228/5734)를 참조하십시오.

### CDN은 서브 도메인 이름 액세스를 지원합니까?
CDN은 현재 서브 도메인 이름 액세스를 지원하지만 인증이 필요합니다. Tencent Cloud에서 제공한 인증 파일을 웹 사이트의 디렉터리에 업로드하고 인증이 완료되면 서브 도메인 이름에 액세스할 수 있습니다.
이 외
1. `* .test.com`과 같은 서브 도메인 이름이 Tencent Cloud에 액세스되어 있다면 해당 서브 도메인 이름을 지원하지 않는 임의의 도메인 이름으로 다른 계정에서 액세스합니다.
2. 서브 도메인 이름`* .test.com`에 액세스되었다면 이 계정에서`* .path.test.com`과 같은 형식의 서브 도메인 이름에 액세스할 수 없습니다.

### CDN 구성에는 시간이 얼마나 걸립니까?
CDN 구성 시간은 30분을 넘지 않으므로 설정 후 대기 시간이 30분을 초과하였다면 [티켓 제출](https://console.cloud.tencent.com/workorder/category) 로 문의하여 도움을 받으십시오.

###  CDN의 도메인 이름에 액세스하려면 어떤 요구 사항이 있습니까?
CDN 액세스를 통해 가속화된 도메인 이름을 생성하려면 공업정보화부에 ICP비안 등록을 완료해야 하며 원본 서버의 서비스 콘텐츠는 합법적이어야 합니다. 그렇지 않으면 도메인 이름에 액세스할 수 없습니다.

### 원본 서버 IP를 여러 개 구성할 수 있습니까?
여러 개의 원본 서버 IP를 구성할 수 있습니다. 여러 개의 IP를 구성하였다면 CDN은 원본 요청할 때 무작위로 IP 중 1개에 액세스합니다. 1개 IP가 원본 가져오기 실패 횟수가  임계 값을 초과하면 해당 IP는 기본 300초 동안 격리되어 원본 가져오기 요청을 진행하지 않습니다.

### 도메인 이름이 CDN에 액세스한 후 어떻게 CNAME을 바인딩합니까?
[CNAME 구성](https://intl.cloud.tencent.com/doc/product/228/3121)의 작업 설명을 참조하여 DNS 공급자에 CNAME을 바인딩 할 수 있습니다.

### 도메인 이름은 종료할 수 있고 삭제할 수 없습니까?
사용자가 협력자인지 확인하십시오. 협력자의 작업 권한은 CDN 서비스 작성자가 구성하므로 작성자가 해당 권한을 지정하지 않으면 협력자가 작업할 수 없습니다. 권한을 받았지만 해당 작업을 실행할 수 없다면 [티켓 제출](https://console.cloud.tencent.com/workorder/category)하여 유지보수 담당자가 처리하도록 합니다.

### 가속 서비스 종료 후 도메인 이름 구성은 저장됩니까?
종료된 도메인 이름 구성은 저장되지만 가속 서비스는 제공하지 않으며 이 시점에서 사용자 요청으로 직접 원본 가져오기를 진행합니다.

### 가속화된 도메인 이름을 삭제한 후 도메인 이름 구성은 저장됩니까?
도메인 이름이 삭제되면 해당 구성은 저장되지 않습니다.

### 가속 서비스를 종료하려면 어떻게해야합니까?
CDN 콘솔에서 가속 서비스를 종료할 수 있습니다. 자세한 내용은 [가속 종료](https://intl.cloud.tencent.com/document/product/228/5736#.E5.85.B3.E9.97.AD.E5.8A.A0.E9.80.9F)를 참조하십시오.

### 가속 도메인 이름은 어떻게 삭제합니까?
CDN 콘솔에서 가속 도메인 이름을 삭제할 수 있습니다. 자세한 내용은 [가속 도메인 이름 삭제](https://intl.cloud.tencent.com/document/product/228/5736#.E5.88.A0.E9.99A4.E5.8A.A0.E9.80.9F.E5.9F.9F.E5.90.8D)를 참조하십시오.

### 도메인 이름을 차단 해제하는 방법은 무엇입니까?
[티켓 제출](https://console.cloud.tencent.com/workorder/category)이 필요하며 전문적인 유지보수 담당자가 처리합니다.
